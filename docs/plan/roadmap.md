# l1nk Development Roadmap

l1nk の開発を以下のフェーズに分けて進める。

## Phase 1: MVP (Minimum Viable Product) - 基本機能の実装
まずは Wiki としての最小機能を実装し、単一ユーザーがメモを記録・閲覧できるようにする。

- [ ] **バックエンド: 基本 API の実装**
  - [ ] ページの CRUD (Hono + D1)
  - [ ] 2-hop link の計算ロジックの実装
- [ ] **フロントエンド: 基本 UI の構築**
  - [ ] Scrapbox ライクなシンプルなエディタ (ブラケットリンクのパース)
  - [ ] ページ詳細表示とリンクのナビゲーション
  - [ ] ページ下部の 2-hop link セクションの表示
- [ ] **認証基盤の完成**
  - [ ] Google/GitHub ログインの実装完了
  - [ ] セッション管理 (KV) の動作確認

## Phase 2: Real-time & Synchronization - リアルタイム共同編集
複数人での編集を可能にし、データの整合性を保つための基盤を構築する。

- [ ] **Durable Objects + Yjs の統合**
  - [ ] 各ページに対応する DO の実装
  - [ ] WebSocket 経由での Yjs 同期プロトコルの実装
- [ ] **非同期データ同期 (DO -> D1)**
  - [ ] DO での変更を検知し、数秒後に D1 へメタデータ（タイトル、リンク、テキストスナップショット）を書き戻す仕組み

## Phase 3: Search & Discovery - 検索と発見
蓄積された知識を効率よく活用するための機能を実装する。

- [ ] **全文検索の実装**
  - [ ] `page_snapshots` テーブルを使用した検索 API
  - [ ] フロントエンドの検索バー UI
- [ ] **ページ一覧・最近の更新**
  - [ ] 更新日時順のカード型・リスト型表示

## Phase 4: Assets & Multimedia - アセット管理
画像などのバイナリデータを扱えるようにする。

- [ ] **R2 ストレージの統合**
  - [ ] 画像アップロード API (Presigned URL 方式)
  - [ ] エディタ内での画像プレビュー表示

## Phase 5: Refinement - 洗練と拡張
実用性を高めるための調整を行う。

- [ ] **UI/UX のブラッシュアップ**
  - [ ] Tailwind CSS を使用した洗練されたデザイン
  - [ ] ダークモード対応
- [ ] **マルチプロジェクト対応**
  - [ ] ユーザーが複数の Wiki (Project) を作成・参加できる仕組み
